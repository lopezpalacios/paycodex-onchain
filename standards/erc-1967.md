# ERC-1967 — Proxy Storage Slots

Standardized storage slots for proxy contract pointers (impl, admin, beacon). Enables upgradeable contracts.

## Why

- Storage slot collisions are a known proxy bug
- ERC-1967 reserves specific slots, all proxies use same convention
- Tools (Etherscan, Tenderly) can read impl + admin without convention specific to a project

## Cash mgmt relevance

- Bank-issued tokens upgradeable via UUPS or Transparent proxy
- ERC-1967 = lower bound on safety
- Pair with timelock + multisig for governance
