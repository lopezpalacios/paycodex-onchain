# ERC-2612 — Permit

Gasless approval via off-chain signature. Eliminates separate approve transaction.

## Why

ERC-20 default flow: approve (tx 1) → transferFrom (tx 2). Two gas costs, two confirmations. Permit collapses to one.

## Interface

```solidity
function permit(
    address owner,
    address spender,
    uint256 value,
    uint256 deadline,
    uint8 v, bytes32 r, bytes32 s
) external;

function nonces(address owner) external view returns (uint256);
function DOMAIN_SEPARATOR() external view returns (bytes32);
```

## Flow

1. Owner signs typed message off-chain (EIP-712)
2. Spender submits message + transferFrom in single tx
3. Owner pays no gas

## Cash mgmt use

- Treasury approves disbursement service to pull funds — no gas tx from treasury
- Recurring authorization (mandate-equivalent) — see [[../use-cases/014-direct-debit-mandate]]
- Used heavily in DEX trades

## Tokens supporting permit

- USDC (since 2022)
- DAI (different signature, predates EIP-2612)
- Most modern stablecoins
