# ERC-4337 — Account Abstraction

Smart contract wallets without consensus changes. Bundlers + paymasters + entry-point contract enable smart accounts.

## Components

- **EntryPoint** — singleton, validates + executes UserOperations
- **Smart Account** — wallet contract owned by user
- **Bundler** — RPC service that bundles UserOps into eth_sendTransaction
- **Paymaster** — sponsors gas (corp pays for treasury txs)

## Cash mgmt use

- **Treasury smart wallet** — multi-sig, spending limits, role-based access on-chain
- **Recurring payment** — session keys auto-pay specific recipients
- **Gasless** — bank pays gas for corporate customer (paymaster)
- **Social recovery** — lose key, recover via approved guardians (replaces seed phrase)
- **Conditional spending** — daily limits, IP restrictions enforced on-chain

## Interface (UserOperation)

```solidity
struct UserOperation {
    address sender;
    uint256 nonce;
    bytes initCode;
    bytes callData;
    uint256 callGasLimit;
    uint256 verificationGasLimit;
    uint256 preVerificationGas;
    uint256 maxFeePerGas;
    uint256 maxPriorityFeePerGas;
    bytes paymasterAndData;
    bytes signature;
}
```
