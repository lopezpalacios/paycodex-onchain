# ERC-1155 — Multi-Token

Combine fungible + non-fungible in single contract. Batch operations native.

## Cash mgmt use

- **Invoice batches** — many invoices same buyer = batched NFT-likes with quantity
- **Voucher / coupon** — bond coupons as semi-fungible tokens (each coupon date = a tokenId, fungible within date)
- **Trade finance documents** — set of related docs as one batch

## Why over ERC-721

- Cheaper batch transfers
- Single contract handles many token types
- Better for AR portfolios
