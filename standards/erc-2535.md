# ERC-2535 — Diamond Pattern

Modular contract architecture. One Diamond proxy delegates to many Facets. Add / remove / replace facets.

## Why

- Bypass 24KB contract size limit
- Upgrade specific functions, not whole contract
- Modular code organization

## Cash mgmt use

- Tokenization platform with many features (mint, burn, freeze, transfer, compliance, governance) split into facets
- Add new feature without redeploy
- Replace compliance facet on reg change

## Components

- **Diamond** — single address users interact with
- **Facets** — feature contracts (logic only)
- **DiamondCut** — admin function to add / replace / remove facets
- **Loupe** — introspection (which facets, which functions)
